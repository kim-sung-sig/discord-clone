Discord LLM Wiki 스킬 분리 리포트

프로젝트 단위 스킬 재구성 및 에이전트팀 설계 결과

핵심 요약

개별 스킬을 사람이 골라 쓰는 구조를 7개 프로젝트-local 스킬과 blueprint 중심 development loop로 감쌌습니다.

  • 성과: loop coordinator, 라우팅, raw-to-docs 정책, 지식 그래프 합성, 구현 청사진, 자연어 변경 영향 분석, accepted knowledge publishing을 독립 책임으로 나눴습니다.
  • 검증: `quick_validate.py`로 7개 LLM Wiki 스킬이 모두 유효함을 확인했습니다.
  • 위치: 사용자 단위가 아니라 `C:\git\discord\.codex\skills` 아래 프로젝트 단위 스킬로만 구성했습니다.

I. 요청 의도 해석

요청의 핵심은 “Karpathy식 LLM Wiki 운영 방식”을 이 저장소의 개발 루프에 넣는 것이었습니다. 단순히 위키를 읽는 것이 아니라, 흩어진 정보를 도메인 언어와 관계 그래프로 키우고, 그 그래프에서 구현 글을 만들며, 자연어 요구사항이 바뀌면 코드 영향까지 추적하는 체계가 필요했습니다.

따라서 `discord-wiki-before-development` 하나에 모든 기능을 넣지 않고, 목적이 다른 기능을 별도 스킬로 분리하는 쪽이 더 타당하다고 판단했습니다.

II. Before & After

기존 구조개별 wiki 스킬을 사람이 골라 호출하는 구조였습니다.
변경 구조`discord-llmwiki-development-loop`가 사용자-facing entrypoint가 되고, primitive 스킬들은 루프 내부 단계가 되었습니다.
이전
개발 전 위키 선택 → 구현 → 개발 후 위키 반영
↓
현재
context intake → raw-to-docs → synthesis → blueprint → implementation → blueprint-vs-implementation review loop → verification → accepted knowledge publishing

III. 작성된 프로젝트-local 스킬

discord-llmwiki-development-loop Loop

일반 개발 작업의 사용자-facing entrypoint입니다. context intake부터 accepted knowledge publishing까지 전체 루프를 조율합니다.

discord-wiki-before-development 라우터

개발 시작 전 `index.md`와 관련 wiki page만 읽고, 필요한 경우 다른 LLM Wiki 스킬로 넘기는 경량 진입점입니다.

discord-raw-to-docs-policy Raw To Docs

Karpathy식 LLM Wiki 정책에 따라 raw source와 synthesis docs를 분리하고, index/log/cross-link/contradiction 규칙을 적용합니다.

discord-llm-wiki-synthesis 지식 그래프

코드, 위키, 설문, 노트를 도메인 언어, tier/layer map, structure graph, behavior flow로 합성합니다.

discord-implementation-blueprint 구현 글

구현 전에 capability, 참여 코드, 클래스/컴포넌트 관계, invariant, verification gate를 포함한 구현 청사진을 작성합니다.

discord-wiki-change-impact 변경 영향

위키 문장, 도메인 언어, 로드맵, 사용자 정정이 바뀌었을 때 graph node와 repository path, 테스트 영향을 추적합니다.

discord-wiki-publish-accepted-knowledge Accepted Publish

blueprint-vs-implementation review와 verification이 통과한 accepted knowledge만 외부 Obsidian Wiki에 반영합니다.

IV. 에이전트팀 구성

에이전트주 스킬책임
Loop Coordinator Agent`discord-llmwiki-development-loop`전체 blueprint 중심 개발 루프를 조율합니다.
Wiki Router Agent`discord-wiki-before-development`작업 도메인을 분류하고 최소 위키 컨텍스트를 선택합니다.
Raw-to-Docs Steward Agent`discord-raw-to-docs-policy`raw source를 durable docs로 승격하기 전에 Karpathy식 LLM Wiki 정책을 적용합니다.
Knowledge Graph Curator Agent`discord-llm-wiki-synthesis`도메인 언어, 관계 그래프, tier/layer map을 정리합니다.
Implementation Architect Agent`discord-implementation-blueprint`구현 전에 상세 구현 글과 검증 기준을 작성합니다.
Change Impact Analyst Agent`discord-wiki-change-impact`자연어 변경을 코드 영향 분석으로 변환합니다.
Implementation Agent작업별 개발 스킬청사진과 실제 코드 기준으로 구현합니다.
Verification AgentQA/test gate청사진이 지정한 테스트와 계약 검증을 실행합니다.
Wiki Publisher Agent`discord-wiki-publish-accepted-knowledge`리뷰와 검증을 통과한 accepted knowledge만 위키에 발행합니다.
