PR 설명 초안

## Summary
- Renew message publishing around MessageAuthor, MessageTarget, MessageContent, MessageMentionTarget, and client-generated IdempotencyKey.
- Route create/list/edit/delete/pin through message use case/read boundaries.
- Replace the PostgreSQL message snapshot/loadAll path with JDBC persistence ports and explicit relational message/idempotency/outbox tables.
- Add transactional message publication outbox, relay claim lease, retry backoff, dead-letter listing, and replay-to-relay-queue operator flow.
- Include related T171 security hardening already present on this branch: trusted proxy policy, security dashboard MFA/operator audit updates, and T190 gate wiring fixes.

## Branch Note
This PR is opened from feature/t02-guild-channel-permission, but the real current slice is T172 message publishing renewal. The branch name is historical and should not be used as the PR scope.
