# T06 Nuxt Discord Shell Plan

작성일: 2026-05-14  
PDCA Phase: Plan  
Slice: T06 nuxt discord shell

## Executive Summary

| 관점 | 내용 |
| --- | --- |
| Problem | shell 컴포넌트는 존재하지만 API/Gateway 연결 seam, story 자산, visual smoke 증거가 없어 UI 회귀를 체계적으로 막기 어렵다. |
| Solution | Nuxt shell을 responsive/a11y 기준으로 정리하고, REST/Gateway client contract와 story/visual smoke harness를 추가한다. |
| Function UX Effect | login -> guild -> channel -> send message 흐름과 desktop/mobile visual smoke를 테스트 아티팩트로 확인할 수 있다. |
| Core Value | T07 친구/DM부터 UI 추가가 기존 shell layout, keyboard, API seam을 깨지 않는 기준선을 갖는다. |

## Scope

- server rail, channel sidebar, chat viewport, member sidebar, user panel layout hardening
- responsive mobile/tablet/desktop checks
- API/Gateway client seam for shell hydration and event apply
- component story files for shell primitives
- component tests and Playwright login -> app -> channel -> send message
- visual smoke screenshot generation through Playwright

## Out Of Scope

- Full Storybook runtime installation
- Live backend boot orchestration from Playwright
- Production WebSocket browser transport
- Pixel-perfect visual regression diffing

## Success Criteria

- story files exist for shell, chat viewport, gateway status, invite modal
- component tests cover keyboard/focus and API/Gateway contract seam
- Playwright covers login -> app -> guild -> channel -> send message
- Playwright writes desktop and mobile visual smoke screenshots
- responsive layout keeps core landmarks visible or intentionally collapsed

## Failure Criteria

- mobile layout hides all navigation without alternate access
- API/Gateway seam remains pure static seed state
- story files rot without tests
- visual smoke cannot be regenerated by a deterministic command
