# T13 Voice/SFU Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build voice channel join/leave signaling, permission-gated SFU token skeleton, voice state controls, and UI smoke coverage.

**Architecture:** Backend owns voice authorization, in-memory participant state, deterministic token issuing, and gateway event projections. Nuxt renders a voice operations panel wired to Pinia state/actions.

**Tech Stack:** Java 21, Spring Boot, JUnit/MockMvc, Nuxt 4, Pinia, Vitest, Playwright.
