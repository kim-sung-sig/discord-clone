# T162 Strict Workspace Native NPM SBOM Cleanup Design

Date: 2026-05-20
PDCA Phase: Design
Slice: T162 Strict Workspace Native NPM SBOM Cleanup

## Dependency Layout

The strict workspace SBOM failure was caused by peer resolution:

```text
@nuxt/cli -> @bomb.sh/tab -> optional peer commander@^13.1.0
editorconfig -> commander@^10.0.0
```

With no root `commander@13`, npm resolved the optional peer to the root `commander@10.0.1`, which made Arborist mark the tree invalid and stopped `npm sbom --workspaces`.

The stable layout is:

```text
root -> commander@13.1.0
editorconfig -> commander@10.0.1 nested under editorconfig
@bomb.sh/tab -> commander@13.1.0 deduped from root
```

## Gate Behavior

- `frontend-sbom.npm.raw.json` is generated by `npm sbom --sbom-format cyclonedx --workspaces`.
- Any non-zero native workspace SBOM exit code stops the security gate.
- `frontend-sbom.json` is copied from the native workspace SBOM artifact.
- `frontend-sbom.npm.lockfile.json` remains as secondary package-lock-only evidence.
- The gate no longer invokes `qa/security-frontend-sbom.mjs`.

## Security Review

Failing closed is preferable for SBOM evidence because a fallback generator can diverge from npm's actual workspace dependency model. Keeping native npm output as the source of truth makes CI evidence easier to audit.
