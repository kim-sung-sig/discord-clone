## Security Notes

- Gateway session registry:
  - Redis shared session metadata stores only session id, user id, guild id set, acknowledgement timestamp, closed flag, and last delivered sequence.
  - Redis storage is per-session key plus index set: `gateway:sessions` and `gateway:sessions:<sessionId>`.
  - `DISCORD_GATEWAY_SESSION_TTL_SECONDS=86400` controls session key TTL; save refreshes TTL and enumeration prunes stale index members.
  - It must not store access tokens, refresh tokens, LiveKit JWTs, request headers, signed URLs, or passwords.

- Redis Gateway stream fanout:
  - Consumer-group processing now reads pending entries before new entries and ACKs handled records.
  - Effective Redis consumer groups are node-scoped as `<configured-prefix>:<node-id>` so fanout is broadcast rather
    than load-balanced across gateway nodes.
  - Resume and poll re-register current visible guild/channel subscriptions before delivery calculation.
  - Same-node stream records with `sourceNodeId` equal to the current node are decoded and ACKed without duplicate listener delivery because local publish already notified local listeners.
  - Malformed records are still dead-lettered before same-node suppression because suppression applies only after successful decode.
  - Stream metrics are aggregate-only: processed, ACKed, decode failures, read failures, and trim count.
  - Do not add raw payload excerpts, stream values, access tokens, LiveKit JWTs, or signed URLs to these metrics.
  - Redis DLQ records are metadata-only and include reason, nodes, event ID/type, stream, record ID, size, and
    `messageSha256Prefix`; they must not contain raw payloads or exception text.

- Operator tokens:
  - `NUXT_SECURITY_DASHBOARD_TOKEN` is now a bootstrap secret.
  - `/api/security/operator-token/exchange` returns the raw issued `sdo_...` token once.
  - Issued tokens are stored as SHA-256 hashes only.
  - UI stores the issued token in `sessionStorage` but renders only expiry.
  - T169 adds Postgres persistence for issued token hashes and audit rows.

- CSP alert acknowledgement:
  - Requires current alert fingerprint.
  - Stale fingerprints return 409.
  - Missing fingerprints return 400.
  - Reasons are required and sanitized.
  - Snooze is bounded to 1-1440 minutes.

- CSP rate-limit diagnostics:
  - UI/API expose source labels, trusted proxy flags, header presence flags, and 12-char hash prefix only.
  - No raw IP or raw forwarded header is exposed.

- Real backend browser smoke:
  - Root gate is `npm run e2e:real-backend`.
  - Local default backend URL is `http://127.0.0.1:18080` to avoid common 8080 conflicts.
  - Local default Postgres URL is `jdbc:postgresql://127.0.0.1:15432/discord` to match Compose.
  - The harness disables Redis health only for this Postgres/browser gate; Redis remains covered by central Redis smoke.
  - The Playwright flow asserts access tokens are not persisted in browser storage or cookies.
